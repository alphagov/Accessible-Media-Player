Accessible Media Player

Dependencies

Basic

The following libraries are required for operation of the Nomensa accessible media player:

  • jQuery core (version 1.3.2+)
  • jQuery UI (version 1.7.3+)

Youtube

No further libraries are required to make the nomensa accessible media player work with the youtube player.

JW Player

In addition to the files mentioned in the "Basic" section above the jwplayer requires the following files:

  • jwplayer.swf (version 5.x) - This is the actual flash based media player. You will be responsible for purchasing and licensing your own copy of JW Player.
  • jwplayer.js. (This file is needed to create the player manager instance. It should be distributed with the jwplayer package)

Initialisation

Nomensas' accessible media player is configured to handle youtube content by default. We currently have config files that make it possible to play audio and video using jwplayer.

Player options

Unique options

These are normally unique to each player:

  • id - (ytplayer)

    A unique id for this player. It is essential that each call to generate a new player passes in a specific new id. The player manager class will use this id for managing player instances on the page.

  • url - (http://www.youtube.com/apiplayer?enablejsapi=1&playerapiid=)

    The path to the player file. The Youtube player instance is returned from the url above. JWPlayer makes use of a .swf file. If a file is used instead this property should be set so that it points to the swf file. For example: "path/to/player.swf"

  • media - (8LiQ-bLJaM4)

    The media file you want the player to load when generated. For YouTube players this is the 11 character unique id at the end of the URL of a YouTube video. For JW players it is the relative path to the media file you want the player to load.

  • captions - (null)

    This should be a path to an xml based captions file. The mediaplayer will get these captions using an ajax call and will synchronise them if possible. Note that this is an experimental feature. Although it should work independantly of the type of video player used it is a purely javascript implementation.

  • image - ()

    Thumbnail image URL that appears before the media is played (this is not supported by the Youtube API)

Generic options

These options will normally be set once for all players:

  • flashWidth - (100%)

    The width of the flash player within the container. This changes the width of the visual screen of the flash player rather than the container. This defaults to 100%, but can be changed to other values such as px or different %'s.

  • flashHeight - (220)

    The height of the flash player within the container. The default value is 600px, but different values can be passed in using the option flashHeight:'200px'.

  • playerStyles - (javascript object - {})

    This is an object containing a mapping of styles that will be applied to the player container. You can add any number of styles to the player. For example "playerStyles : {'height': '100%', 'width': '100%', 'color':'red'}" will create a player container of 100% width and height with red text.

  • captionsOn - (false)

    This is a simple boolean flag indicating whether or not to display captions by default (if a captions file has been provided).

  • use_html5 - (true)

    If this option is set to true, HTML5 version of the player will be used if the browser in use supports it. If the browser in use does not support the html5 video element or the type of video being played it will fall back to use flash. Note that this does not currently work with the Youtube player. Support for any given type of video is evaluated based on the level of support estimated by the browser. This uses the HTML5 video/audio 'canPlayType(videoType)' call.

Advanced options

The defaults will normally be sufficient. We would recommend you only change these if you know what you are doing:

  • repeat - (false)

    Whether or not to loop the video.

  • slider_timeout - (350)

    The time between sebsequent calls to update the timer widget in milliseconds

  • flashContainerName - (player)

  • flashContainer - (span)

    Element used for Flash container

  • playerContainer - (span)

    Element used for player container

  • player_skip - (10)

    Amount (in seconds) the rewind and forward buttons skip

  • volume_step - (10)

    The value (in percent) that the volume will be increased/decreased by when it is changed.

  • buttons - (javascript object - {})

    A mapping of buttons to show in the player.

    For example:

    buttons : {
	forward: true,
	rewind: true,
	toggle: true
}

    This mapping will ensure that the fast-forward, rewind and a play/pause button is provided. Note that if 'toggle' is set to true only one button will be provided. This will be used to toggle between playing and paused states. If this is set to false both a play and a pause button will be provided.

  • player - (null)

    This is purely a placeholder. It will eventually contain a reference to the player API (whichever one is being used). If this property is overwritten when the player is instantiated it is likely to cause problems.

  • swfCallback - (null)

    SWFObject allows us to attach a callback to the swf.embed option. This allows us to call a function, method when the flash component has loaded. This is required for the correct operation of some players.
